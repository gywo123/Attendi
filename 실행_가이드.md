# Attendi 실행 가이드

## 스택

- Frontend: React + Vite + PWA
- Backend: Node.js Express
- DB: SQLite (`node:sqlite`)
- Auth: Google OAuth + 교사 이메일 로그인
- QR: `qrcode` + `html5-qrcode`
- API: REST

## 1. 백엔드 실행

```bash
cd server
node src/index.js
```

또는:

```bash
cd server
npm run dev
```

백엔드 주소:

```text
http://localhost:4000/api
```

헬스체크:

```text
http://localhost:4000/api/health
```

## 2. 프론트엔드 실행

```bash
cd front
npm run dev
```

프론트 주소:

```text
http://localhost:5173
```

## 3. Google OAuth 설정

Google Cloud Console의 승인된 리디렉션 URI:

```text
http://localhost:4000/api/auth/google/callback
```

승인된 JavaScript 원본:

```text
http://localhost:5173
http://127.0.0.1:5173
```

루트 폴더의 `client_secret*.json` 파일은 백엔드가 자동으로 읽는다.

이 파일은 `.gitignore`에 포함되어 있으므로 Git에 올리면 안 된다.

## 4. 초기 계정

관리자:

```text
admin@school.kr / 1234
```

교사, 학생, 인식 기기 토큰은 관리자 화면에서 추가하거나 승인한다.

## 5. 분리 배포 설정

프론트는 Cloudflare Pages에 `front` 폴더 기준으로 배포한다.

```text
Build command: npm run build
Build output: dist
Environment variable: VITE_API_BASE_URL=https://백엔드주소/api
```

백엔드는 `server` 폴더 기준으로 Node.js 서버 호스팅(Render, Railway 등)에 배포한다.

```text
Start command: npm start
Environment variable: CLIENT_URL=https://프론트주소
Google redirect URI: https://백엔드주소/api/auth/google/callback
```

## 6. 현재 구현된 흐름

- 교사 이메일 로그인
- 기기 토큰 로그인
- SQLite 초기 데이터 자동 생성
- 학생 QR 토큰 발급 API
- QR 토큰 검증 API
- 출석 기록 저장
- 출석 요약 조회
- PWA manifest 및 service worker 등록
- 실제 QR 이미지 생성
- 카메라 QR 스캔 라이브러리 연결

## 7. 개발용 GPS 설정

현재 백엔드는 로컬 개발을 위해 기본값으로 `DEV_BYPASS_LOCATION=true`를 사용한다.

실제 GPS 반경 검증을 테스트하려면 `server/.env`를 만들고 다음처럼 설정한다.

```text
DEV_BYPASS_LOCATION=false
SCHOOL_LATITUDE=학교_위도
SCHOOL_LONGITUDE=학교_경도
SCHOOL_RADIUS_METERS=100
```
